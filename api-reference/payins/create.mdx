---
title: 'Criar Cobrança'
api: 'POST /payin'
description: 'Crie cobranças via Pix, Boleto ou Cartão de Crédito.'
---

Este é o endpoint principal da nossa API. Aqui você pode gerar cobranças instantâneas para seus clientes.

## Body

<ParamField body="paymentMethod" type="string" required>
  Método de pagamento.
  Valores aceitos: `PIX`, `CREDIT_CARD`, `BOLETO`.
</ParamField>

<ParamField body="amount" type="integer" required>
  Valor da transação em **centavos**.
  Exemplo: R$ 10,00 deve ser enviado como `1000`.
</ParamField>

<ParamField body="referenceId" type="string" required>
  O ID do pedido no **seu** sistema (ex: "pedido_123").
  Usaremos esse ID para comunicar atualizações via webhook.
</ParamField>

<ParamField body="payerIp" type="string" required>
  Endereço IP do cliente pagador (IPv4 ou IPv6).
</ParamField>

<ParamField body="isPhysicalProduct" type="boolean" required>
  `true` se houver entrega física, `false` para produtos digitais.
</ParamField>

<ParamField body="webhookUrl" type="string">
  URL onde você deseja receber as notificações de status (Webhooks).
</ParamField>

<Note>
  O objeto `card` abaixo só é obrigatório se o `paymentMethod` for `CREDIT_CARD`.
</Note>

<ParamField body="card" type="object">
  <Expandable title="Propriedades do Cartão">
    <ParamField body="holderName" type="string" required>Nome como impresso no cartão.</ParamField>
    <ParamField body="number" type="string" required>Número do cartão (somente dígitos).</ParamField>
    <ParamField body="expirationMonth" type="string" required>Mês de expiração (2 dígitos, ex: "12").</ParamField>
    <ParamField body="expirationYear" type="string" required>Ano de expiração (4 dígitos, ex: "2030").</ParamField>
    <ParamField body="cvv" type="string" required>Código de segurança (CVV).</ParamField>
    <ParamField body="installments" type="integer" required>Número de parcelas (1 a 12).</ParamField>
    <ParamField body="token" type="string">Token do cartão para compras futuras (one-click).</ParamField>
  </Expandable>
</ParamField>

### Objeto Customer

Dados do cliente são obrigatórios para emissão de notas e análise de fraude.

<ParamField body="customer" type="object" required>
  <Expandable title="Propriedades do Cliente">
    <ParamField body="name" type="string" required>Nome completo.</ParamField>
    <ParamField body="document" type="string" required>CPF ou CNPJ (somente números).</ParamField>
    <ParamField body="email" type="string" required>E-mail válido.</ParamField>
    <ParamField body="phone" type="string" required>Telefone com DDD (somente números).</ParamField>
    <ParamField body="address" type="object" required>
        <Expandable title="Endereço">
            <ParamField body="street" type="string" required>Rua / Logradouro.</ParamField>
            <ParamField body="number" type="string" required>Número.</ParamField>
            <ParamField body="complement" type="string">Complemento (Apto, Bloco).</ParamField>
            <ParamField body="zipCode" type="string" required>CEP (somente números).</ParamField>
            <ParamField body="city" type="string" required>Cidade.</ParamField>
            <ParamField body="state" type="string" required>Estado (Sigla, ex: SP).</ParamField>
        </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

### Objeto Items

Lista de itens que compõem o pedido.

<ParamField body="items" type="array" required>
  <Expandable title="Item do Pedido">
    <ParamField body="title" type="string" required>Nome do produto/serviço.</ParamField>
    <ParamField body="quantity" type="integer" required>Quantidade (mínimo 1).</ParamField>
    <ParamField body="unitPrice" type="integer" required>Preço unitário em centavos.</ParamField>
    <ParamField body="description" type="string">Descrição detalhada (opcional).</ParamField>
  </Expandable>
</ParamField>
